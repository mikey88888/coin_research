# AGENTS.md - coin_research

这个仓库允许自动迭代，但必须收敛。

## 工作边界

- 只在本仓库内工作：`/home/thinkpad/quant/coin_research`
- 不要修改仓库外文件
- 不要改 OpenClaw 全局配置
- 不要执行破坏性命令
- 不要自动 push 远端，除非当前明确要求

## 2 小时迭代规则

每轮只做 **1 个小优化**，优先级如下：

1. 提升研究效率
2. 提升测试/可验证性
3. 提升数据同步可靠性
4. 提升网页可用性
5. 提升文档清晰度

典型允许动作：

- 增补测试
- 修小 bug
- 小范围重构
- 完善 README / 使用说明
- 加轻量 smoke 脚本
- 改善错误提示、参数校验、日志

典型禁止动作：

- 大规模架构重写
- 大版本依赖升级
- 破坏性 schema 变更
- 未验证就批量改多个模块
- 自动发布、自动部署、自动推远端

## 执行流程

1. 先读仓库现状，提出 **1 个最小但有价值的改进点**
2. 改动前说明目标
3. 只做与该目标直接相关的最小改动
4. 改完后运行：

```bash
uv run pytest -q
```

5. 若测试失败：停止继续扩写，先修当前问题或汇报阻塞
6. 若测试通过：总结改了什么、为什么值得

## 变更规模控制

- 单轮尽量只动少量文件
- 除非必要，不同时改数据层、回测层、网页层三块
- 优先做可验证、可回滚的小修

## 输出要求

每轮结束时给出：

- 本轮优化点
- 改动文件
- 测试结果
- 风险/后续建议

风格要求：冷静、干练、少废话。
